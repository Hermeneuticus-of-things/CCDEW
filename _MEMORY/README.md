# Universal Memory Format

## Overview

This directory contains the **Universal Memory** for the CCDEW project. All entries are stored as plain Markdown (`.md`) files with YAML frontmatter, making them:

- **Human-readable** — Open in any text editor
- **LLM-compatible** — Any AI can parse Markdown + YAML natively
- **Cross-platform** — Works with OpenCode Desktop, Claude Code, Cursor, VS Code, etc.
- **Future-proof** — Plain text, no vendor lock-in

## Structure

```
_MEMORY/
├── L0-sensory/     # Immediate input (1 hour, raw logs)
├── L1-working/     # Active tasks & goals (3 days)
├── L2-episodic/    # Session snapshots (14 days)
├── L3-semantic/    # Concepts & patterns (30 days)
└── L4-identity/    # Permanent project knowledge
```

## File Format

Each file follows this structure:

```markdown
---
uid: ccdew-20260523-144500-abc123
project: home-think-CCDEW
layer: L4
type: identity
created_at: '2026-05-23T14:45:00.000Z'
tags:
  - architecture
  - core
confidence: 1
---

# Title of the Entry

## Context
Description here...

## Details
- Bullet points
- More details

---

*Generated by CCDEW Universal Memory v2.0*
```

## Layer Decay

| Layer | Duration | Purpose |
|-------|----------|---------|
| L0 | 1 hour | Raw sensory input |
| L1 | 3 days | Working memory (tasks, goals) |
| L2 | 14 days | Episodic sessions |
| L3 | 30 days | Semantic concepts & patterns |
| L4 | Permanent | Identity & core knowledge |

## Propagation

Entries flow upward:
```
L0 → L1 → L2 → L3 → L4 → OS-Brain
```

- **L0→L1**: Automatic (repeated insights)
- **L1→L2**: Manual or on session end
- **L2→L3**: Confidence threshold (≥0.6)
- **L3→L4**: High confidence (≥0.85)
- **L4→OS**: Stable knowledge (>1 day old)

## Usage

### Manual (any editor)
Simply create a `.md` file with YAML frontmatter in the appropriate layer directory.

### Programmatic
```bash
node universal-memory.cjs write L1 "Task Title" "Task description..."
node universal-memory.cjs search "keyword"
node universal-memory.cjs list L3
node universal-memory.cjs propagate
```
