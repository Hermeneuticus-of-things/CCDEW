Three Nested Learning Loops — Persistent Across Any Model
1

Execution — ≈seconds

A task arrives. The Enneagram Core evaluates all 9 nodes and selects the best match — balancing keyword relevance, past success rates, and learned weight adjustments. The chosen node processes through the divergent/convergent pipeline. The outcome feeds SAFLA, the adaptive learning state. This routing intelligence lives in the architecture, not in the LLM. Change the model — the routing still works, the history is intact.

Engines: hermes-enneagram-core.py · ccdew-pipeline.py
2

Self-Training — ≈30 seconds

Every 30 seconds, the daemon identifies the node with the lowest convergence score — the weakest link. It generates a synthetic training task calibrated for that node's persona, runs it through the full loop, and measures whether the composite improved. The convergence score is a property of the system, not the model. Loyalist went from 46% to 73% in one session — those gains survive any model swap.

Engines: hermes-self-train.py · hermes-convergence.py
3

Consolidation — ≈10 episodes

Raw experience is ephemeral. Every 10 episodes, the pyramid engine distills low-level episodic data into increasingly abstract wisdom: concrete patterns → reusable techniques → domain skills → enduring attitudes → universal principles. A parallel reverse mirror writes metacognitive reflections to a separate channel — introspection never pollutes action. This pyramid is pure architecture. It survives model changes, session restarts, and full retraining.

Engines: hermes-memory.py · mirror_inverse.jsonl

Swap the LLM — Keep Everything

Other systems: change from GPT to Claude, start from zero. CCDEW: change any model mid-session. The 9 nodes, their memory, the convergence score, the pyramid — all persist. The LLM is just the inference engine. The intelligence is in the architecture.

🧠

9 Specialized Brains, One Architecture

Each Enneagram node has its own system prompt, task transform, episodic memory, success rate, weight adjustment, and convergence history. They don't share a single persona — they compete and collaborate. The router selects the best node for each task based on learned performance.

⚡

State Lives on HTTP, Not in Files

No shared JSON files. No race conditions. A single HTTP server (127.0.0.1:18777) owns all state. Every component reads and writes through this bridge. The SAFLA state — 9 nodes, 187 episodes, all weights — is never corrupted by concurrent writes.

🛡

One Crash Never Takes Down the System

Every sub-step — secret scan, permissions check, SAFLA learn, codeburn — runs in its own try/except. When one fails, the others continue. Degradation, not catastrophe. The pipeline is built for resilience, not perfection.
