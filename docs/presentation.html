The Problem

What's broken in AI agents today

Every AI system struggles with the same four issues. CCDEW was designed from the ground up to solve all of them.

✕

LLMs never improve with use

Same knowledge today = same knowledge tomorrow. Zero growth between sessions. No compounding improvement.

→ Self-training daemon drives convergence 24/7
✕

One-size-fits-all reasoning

Every task gets the same generic persona. No specialized perspective. Code review uses same approach as creative writing.

→ 9 Enneagram nodes with unique personas
✕

No measurable quality

No dashboard. No metric. No way to tell if the system is improving. Blind deployment with no feedback loop.

→ Live convergence score 0–100%, updated 30s
✕

Fragile and insecure

One failure crashes everything. Shell injection vectors in MCP. Race conditions in shared state files.

→ HTTP bridge no race conditions, no exec

The Solution

Three-Loop Learning System

Three nested feedback loops at different cadences. Each loop feeds the next. The system measurably improves with every cycle.

Loop 1 ≈s

Execution

Task in → Enneagram Core (9 nodes) → Pipeline divergent/convergent → Record to SAFLA + episodic memory

hermes-enneagram-core.py ccdew-pipeline.py
Loop 2 30s

Self-Training

Daemon picks weakest node → generates task → runs full cycle → measures convergence delta → repeats

hermes-self-train.py hermes-convergence.py
Loop 3 10ep

Consolidation

Extract patterns → techniques → attitudes → principles. Reverse mirror writes to separate channel.

hermes-memory.py
Task → Enneagram (9) → Pipeline → Learn ↻ Self-Train ↻ Consolidate

When to Use

Use Cases

CCDEW fits best when you need an AI system that actively improves, produces diverse perspectives, and gives you hard metrics.

↗

Self-Improving AI Assistant

Deploy an agent that gets measurably better every day — no manual retuning, no human oversight. The daemon finds and trains the weakest node autonomously.

Best for: production systems that need ongoing improvement
🧠

Multi-Perspective Analysis

Route a problem through 9 cognitive personas. Each Enneagram type produces a different perspective. Converge them into one synthetic verdict.

Best for: complex decisions, code review, strategy
⚙

Quality Gate Automation

Pre-commit checks, pre-push verification, code review, risk assessment. Each step routed through the best-suited cognitive node for maximum accuracy.

Best for: CI/CD pipelines, security audits, compliance
📚

Research & Learning System

Memory pyramid auto-consolidates everything learned. Raw actions (N1) → abstract principles (N6). No knowledge is ever lost between sessions.

Best for: research assistants, knowledge management

Under The Hood

Infrastructure

Every component has a single clear responsibility. No shared files. No race conditions. No shell injection.

HTTP Bridge

127.0.0.1:18777 — single Python process serves all state. Others read/write via HTTP. Zero race conditions.

SAFLA State

Written exclusively by pipeline.py. 9 nodes tracked. 187 episodes. 100% single-writer — no data loss.

MCP Servers

4 active servers. Native http.request in Node.js. No shell exec, no curl, no injection vectors.

Inner Observer

Metacognitive daemon writes consciousness.jsonl (append-only). Tracks confidence, flow, confusion, satisfaction.

# Bridge API endpoints GET /bridge → bridge state snapshot
GET /safla → SAFLA learning state
POST /safla → update SAFLA (single writer)
POST /log → centralized logging
GET /core/select → Enneagram node selection
POST /core/outcome → learn from outcome
GET /convergence → dashboard JSON
GET /health → heartbeat

Clone it. Run it. Watch it improve.

The system starts learning the moment you clone the repo. Python 3.10+ and Node.js 20+ are all you need.

# 1. Clone
git clone https://github.com/Hermeneuticus-of-things/CCDEW.git ~/CCDEW
cd ~/CCDEW

# 2. Start pipeline + HTTP bridge
nohup python3 .claude/helpers/ccdew-pipeline.py serve > /tmp/pipeline.log &

# 3. Start self-training daemon
nohup python3 .claude/helpers/hermes-self-train.py --daemon > /tmp/train.log &

Open OpenCode Desktop in the repo — AGENTS.md loads automatically.
