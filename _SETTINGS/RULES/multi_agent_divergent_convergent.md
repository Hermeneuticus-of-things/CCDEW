# RULES — Multi-Agent Enneagram: Pattern Divergent <> Convergent
## Universal cross-project (anti-omogenizare în swarm-uri)

**Data:** 2026-05-02 — extras din feedback user, generalizat.

## Problema rezolvată

Când spawnez N agenți paraleli pe ACELAȘI task, default-ul lor e să producă **outputuri omogene** — fiecare agent abordează identic, rezultă N copii ale aceluiași output. Echo chamber. N agenți = 1 agent redundant.

**Anti-pattern:** „spawn 5 general-purpose agents pe X" → toți 5 dau aceeași analiză.

## Soluția: Pattern Divergent <> Convergent

### Faza 1 — DIVERGENT (forțat la spawn)

Fiecare agent primește o **ARIPĂ ENNEAGRAMATICĂ DISTINCTĂ** — un alt UNGHI de abordare derivat din 7 noduri fractale:

| Aripa | Direcție de abordare |
|---|---|
| **Zoom-aripa Maha** | Vedere de ansamblu cosmic/sistemic |
| **Zoom-aripa Macro** | Vedere structurală inter-componente |
| **Zoom-aripa Mezzo** | Vedere de proces/flux |
| **Zoom-aripa Micro** | Vedere intra-componentă |
| **Zoom-aripa Nano** | Vedere atomică/microsecundă |
| **Lentilă-aripa stilistică** | Filtru estetic/literar |
| **Lentilă-aripa doctrinară** | Filtru fidelitate sursă |
| **Lentilă-aripa structurală** | Filtru arhitectural |
| **Lentilă-aripa regression** | Filtru ce nu funcționează |
| **Lentilă-aripa memorie** | Filtru continuitate cu trecut |
| **Perspectivă-aripa agent** | Din vedere actor |
| **Perspectivă-aripa observator** | Din vedere extern neutru |
| **Perspectivă-aripa cosmos** | Din vedere sistemic |
| **Perspectivă-aripa Sākṣī** | Din vedere martor non-implicat |
| **Modalitate-aripa descriptivă** | Cum este |
| **Modalitate-aripa normativă** | Cum ar trebui |
| **Modalitate-aripa generativă** | Ce-ar fi dacă |
| **Modalitate-aripa critică** | Ce nu merge |

**Prompt template per agent (faza divergent):**
> „Abordează acest task DOAR prin [ARIPA SPECIFICĂ]. NU folosi alte unghiuri. Output-ul tău e perspectiva [X] pură pe [task]. Alte agenți acoperă alte unghiuri."

### Faza 2 — CONVERGENT (synthesis explicit)

**1 agent dedicat** (Reformer / Sintetizator) primește toate output-urile divergente și produce o sinteză coerentă:

- Identifică convergențe (unde toate aripile sunt de acord)
- Identifică divergențe productive (unde aripile spun lucruri diferite și utile)
- Identifică tensiuni nerezolvate (unde aripile se contrazic și NU se pot reconcilia)
- Output unic care păstrează **diversitatea perspectivelor** dar produce **un verdict integrat**

## Praguri de aplicare

- **2-3 agenți paraleli:** divergent opțional, convergent ușor (poate fi tu însuți)
- **4-6 agenți paraleli:** divergent OBLIGATORIU (alocare aripe distincte explicit)
- **7-10 agenți:** divergent + convergent dedicat OBLIGATORIU
- **>10 agenți:** ierarhic — 3 grupuri × 3-4 agenți, fiecare grup convergent local, apoi convergent global

## Anti-pattern explicite

- ❌ N × general-purpose agents cu același prompt — produc outputuri identice (waste)
- ❌ Convergent fără faza divergent — pierzi profunzimea
- ❌ Divergent fără convergent — produci N rapoarte fragmentate, fără sinteză
- ❌ Aripe duplicate (2 agenți pe Macro) — redundanță
- ❌ Aripe incompatibile cu task-ul (Sākṣī pe debugging cod) — output irelevant

## Aplicabilitate cross-project

| Domain | Aripe productive |
|---|---|
| **Editorial carte** | doctrinar + stilistic + structural + memorie + regression |
| **Code review** | logic + security + performance + maintainability + tests |
| **Architecture decision** | macro + cosmos + normativă + critică + memorie |
| **Bug investigation** | regression + zoom-Nano + observator + critică |
| **Content creation** | generativă + stilistică + sociologică + agent |
| **Research synthesis** | macro + memorie + critică + convergent dedicat |
