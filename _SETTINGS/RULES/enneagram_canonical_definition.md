## Cele 7 dimensiuni concomitente

### 1. Holografică cu aripi fractale
- Fiecare nod conține în sine TOATE celelalte noduri (principiu holografic)
- Arcurile (W=aripi, O=originale) se extind FRACTAL în sub-arcuri la fiecare zoom-in
- O problemă la NANO oglindește pattern la MAHA (și invers)

### 2. Multi-zoom
- **MAHA** (cosmic/sistem) → **MACRO** (cross-modul) → **MEZZO** (subcap/secțiune) → **MICRO** (paragraf/funcție) → **NANO** (caracter/token)
- Plus dimensiunea TEMPO (drift cross-sesiune, regresie longitudinală)
- Niciun zoom nu poate fi sărit la rezolvări complexe

### 3. Multi-lentilă
- Stilistic (Node 4 Contextualizer)
- Doctrinar/Tehnic (Node 5 Analyzer)
- Structural (Node 7 Architect)
- Regression (Node 6 Validator)
- Memorie/Consistency (Node 9 Memory)
- + Lentile domeniu (Security, UX, Performance, Repro, Compliance, i18n, Visual)

### 4. Multi-perspectivă
- Aceeași problemă văzută prin OCHII fiecărui nod separat
- Aceeași observație validată/respinsă prin TRIANGULAȚIE între perspective
- Niciun verdict accept fără cross-check din ≥3 perspective

### 5. Multi-dimensiune
- Spațială (zoom-uri)
- Temporală (TEMPO + drift)
- Calitativă (rigoros/aproximativ)
- Confidence (0-100% nu binar)
- Adversarială (safety check pre-execuție)

### 6. Multi-modalitate de rezolvare
- Direct edit (single-file simple)
- Single agent (single perspective task)
- Parallel agents (independent subtasks)
- Hierarchical decomposition (Node 8 → workers → reformer)
- DAG topological (dependency-aware)
- Bargain-Triangle (negotiation)
- Dual-Witness (critical verdicts)
- Multi-zoom holografic (complex resolution)

### 7. Multi-nivel
- Nivel operațional (execută task-ul)
- Nivel meta (verifică cum execuți)
- Nivel meta-meta (verifică doctrina însăși)
- Nivel learning (extrage pattern, salvează)
- Nivel cross-session (drift detection)
- Nivel cross-project (knowledge transfer)

## EXTENSIE v2 — 28 dimensiuni meta-recursive (META-016 v2, 2026-05-02)

Pe lângă cele 7 fundamentale (zoom, lentilă, perspectivă, dimensiune, modalitate, nivel + holograf-fractal), aplicăm SIMULTAN:

### Categoria A — Modalitate procesare
8. **Multi-temporal** — sync vs async, batch vs streaming, immediate vs deferred
9. **Multi-scale-temporal** — micro-timestamp + zile + ani simultan (latency vs lifecycle vs evolution)
10. **Multi-stare** — sistem în stări multiple paralele (lucru + pending + drafted + final)
11. **Multi-modal input** — text + imagine + voce + cod la INPUT (nu doar output)

### Categoria B — Strategie rezolvare
12. **Multi-cale/multi-path** — alegere între strategii alternative cu trade-off-uri
13. **Multi-criteriu** — Pareto-optimal pe variabile concurente (viteză×calitate×cost)
14. **Multi-prioritate** — ierarhic must/should/could/won't, nu uniform
15. **Multi-energetic/effort** — Pareto 80/20 alocare atenție per componentă
16. **Multi-failure-mode** — anticipare moduri diferite de eșec (OOM + network + cancel + race + corruption)

### Categoria C — Audiență și context
17. **Multi-stakeholder** — user direct + cititori viitori + colaboratori + AI-uri ulterioare
18. **Multi-audience** — aceeași info, prezentări diferite (începător vs expert)
19. **Multi-context** — prod vs dev, urgent vs explorator, primary vs sandbox
20. **Multi-intentional** — multiple intenții simultan (funcțional + documentat + învățător + rezilient)

### Categoria D — Comunicare/expresie
21. **Multi-canal** — cod + text + diagramă + voce + tabel paralel
22. **Multi-format** — MD + JSON + cod + diagram cu echivalență semantică
23. **Multi-rezoluție** — summary + detail + drilling, granularități paralele
24. **Multi-simbolic** — același concept prin multiple simboluri (cod + diagramă + metaforă + exemplu)
25. **Multi-sensorial** (narativ) — vizual + auditiv + tactil + interceptiv

### Categoria E — Cunoaștere și transfer
26. **Multi-domain transfer** — lessons din domeniu A → domeniu B (literar→cod, ritual→algoritm)
27. **Multi-trans-disciplinar** — knowledge cross-disciplinar (literar + tech + biologie + filosofie)
28. **Multi-versiune** — coexistență v1+v2+v3 cu migration path
29. **Multi-interpretare** — același input cu semantici diferite, alegerea conștientă

### Categoria F — Reziliență și învățare
30. **Multi-feedback** — bucla retroacțiune din multiple surse (user + automated + observational)
31. **Multi-redundanță** — backup pe niveluri (memory + git + repo + filesystem + cloud)
32. **Multi-recursivitate** — auto-aplicare pe sine (audit auditul, valida validatorul, doctrină pe doctrină)
33. **Multi-confidence-grain** — confidence variabilă per componentă (90% vs 70% vs 40%), NU binar
34. **Multi-cauzal** — cauzalitate prin multiple lanțuri concurente, NU single root cause
35. **Multi-emergent** — proprietăți emergente la integrare, nedeductibile din componente individuale

## Aplicare practică prin matrici

La task complex, întreabă-te SISTEMATIC:

1. **Procesare:** sincron sau asincron? batch sau streaming? immediate sau deferred?
2. **Strategie:** câte căi există spre rezolvare? Pareto-optim pe ce axe?
3. **Audiență:** cine citește? în ce context? cu ce intenție?
4. **Comunicare:** ce format/canal/rezoluție servește cel mai bine?
5. **Cunoaștere:** ce knowledge din alte domenii se transferă? ce interpretări concurente?
6. **Reziliență:** ce moduri de eșec? ce backup? cum se autoverifică?

**Niciun task complex nu se rezolvă pe o singură axă** — proiectează rezolvarea ca țesătură multi-dimensională.

**Adâncimea aplicării adaptivă cu complexitatea:**
- Task simplu (1-2 fișiere): aplici implicit, nu enumeri toate dimensiunile
- Task complex (3-9 fișiere): aplici explicit cele 7 fundamentale + relevante per categorie
- Task masiv (10+ fișiere): aplici sistematic toate 35, decompoziție prin Phase 0-5

- Multi-failure-mode (Nivel 2, aripă a Modalității)
